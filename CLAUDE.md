## Project Overview

**Tarik Ketupat Matematika** — a classroom math game where a teacher (host) projects a quiz on screen and students join via phone. Two teams (Tim Opor and Tim Rendang) compete by answering math questions; correct answers "pull" a ketupat marker toward their side in a tug-of-war animation.

Phase 1 (current): Fully functional UI with mock state via React Context — no backend yet. The context simulates real-time multiplayer by sharing state across browser tabs.

## Architecture

### State Management

`MockGameContext` (`src/context/MockGameContext.jsx`) is the single source of truth for all game state:

- `roomPin` — 6-digit string generated by `createRoom()`
- `gameStatus` — `'idle'` | `'lobby'` | `'playing'` | `'finished'`
- `gameConfig` — `{ mode: 'penjumlahan'|'pengurangan'|'perkalian', difficulty: 'mudah'|'sedang'|'sulit' }`
- `players` — array of `{ name, team }` where team is `'opor'` or `'rendang'`
- `currentQuestion` / `currentQuestionIndex` — active question from filtered `mockSoal.json`
- `teamScores` — `{ opor: number, rendang: number }`

Actions: `createRoom`, `joinRoom`, `startGame`, `submitAnswer`, `nextQuestion`, `endGame`.

The `MockGameProvider` wraps the entire app in `main.jsx`. All pages consume state via `useMockGame()`.

### Routing

Two flows share the same Context (enabling host + player to coexist in separate tabs):

| Path | Component | Role |
|------|-----------|------|
| `/` | `Home` | Landing — choose Host or Player |
| `/host` | `HostSetup` | Configure mode & difficulty, call `createRoom()` |
| `/host/lobby` | `HostLobby` | Show PIN + QR code, wait for players |
| `/host/game` | `HostGameplay` | Project question + TimerBar + KetupatAnim |
| `/host/result` | `HostResult` | Winner announcement |
| `/play` | `PlayerJoin` | Enter PIN and name |
| `/play/wait` | `PlayerWaiting` | Show team assignment, wait for host |
| `/play/game` | `PlayerGamepad` | 4-button answer controller (AnswerPad) |
| `/play/result` | `PlayerResult` | Win/lose screen on student device |

Pages use `useEffect` navigation guards to redirect if accessed without valid game state (e.g., no `roomPin` redirects to setup).

### Key Components

- **`TimerBar`** (`src/components/host/`) — countdown timer (default 10s), calls `onExpire` when done; color shifts green → amber → red. Keyed to `currentQuestion.id` to reset on question change.
- **`KetupatAnim`** (`src/components/host/`) — horizontal tug-of-war; ketupat marker translates based on `scoreRendang - scoreOpor`, clamped to ±50, with `transition-transform duration-500 ease-out`.
- **`AnswerPad`** (`src/components/player/`) — 2x2 grid of giant buttons; shows correct/wrong color feedback after selection, then disables all buttons. Resets when `currentQuestion.id` changes.

### Data

`src/data/mockSoal.json` — static question bank. Schema:
```json
{ "id", "tipe": "penjumlahan|pengurangan|perkalian", "difficulty": "mudah|sedang|sulit", "soal", "opsi": {"A","B","C","D"}, "jawaban_benar" }
```
Questions are filtered by `gameConfig.mode` (mapped to `tipe`) and `gameConfig.difficulty`. Falls back to all questions if no match.

### Design System

Tailwind CSS v3 with neubrutalism aesthetic. Custom tokens in `tailwind.config.js`:
- `primary` — `#f59f0a` (amber)
- `bg-light` — `#FFFDF5`
- `opor` — `#fbbf24` (amber-400, Tim Opor)
- `rendang` — `#92400e` (amber-900 dark brown, Tim Rendang)
- Font: `Plus Jakarta Sans` (display)
