## Project Overview

**Tarik Ketupat Matematika** — a classroom math game where a teacher (host) projects a quiz on screen and students join via phone. Two teams (Tim Opor and Tim Rendang) compete by answering math questions; correct answers "pull" a ketupat marker toward their side in a tug-of-war animation.

**Phase 2 (current):** Supabase auth (magic link OTP) + Realtime Broadcast for true multiplayer. `GameContext` replaces the old `MockGameContext`.

## Architecture

### State Management

Two contexts wrap the app (`main.jsx`): `AuthProvider` (outer) → `GameProvider` (inner).

**`AuthContext`** (`src/context/AuthContext.jsx`):
- `user` — Supabase auth user or `null`
- `loading` — true during initial session check
- `signIn(email)` — sends magic link OTP
- `signOut()` — clears session

**`GameContext`** (`src/context/GameContext.jsx`) is the single source of truth for all game state:

- `roomPin` — 6-digit string generated by `createRoom()`
- `gameStatus` — `'idle'` | `'lobby'` | `'playing'` | `'finished'`
- `gameConfig` — `{ mode: 'penjumlahan'|'pengurangan'|'perkalian', difficulty: 'mudah'|'sedang'|'sulit' }`
- `players` — array of `{ name, team }` (host side only, updated via Broadcast)
- `currentQuestion` / `currentQuestionIndex` — active question from filtered `mockSoal.json`
- `teamScores` — `{ opor: number, rendang: number }`
- `playerTeam` / `playerName` — player's own team and name (player side only)

Actions: `createRoom`, `joinRoom`, `startGame`, `submitAnswer`, `nextQuestion`, `endGame`.

All actions that modify game state also broadcast events over `supabase.channel('room-{pin}')`.

### Authentication & Routing

All pages except `/` are wrapped in `ProtectedRoute`, which renders `LoginForm` if no active session. After magic link login, users access the game normally.

### Routing

| Path | Component | Role |
|------|-----------|------|
| `/` | `Home` | Landing — choose Host or Player |
| `/host` | `HostSetup` | Configure mode & difficulty, call `createRoom()` |
| `/host/lobby` | `HostLobby` | Show PIN + QR code, wait for players |
| `/host/game` | `HostGameplay` | Project question + TimerBar + KetupatAnim |
| `/host/result` | `HostResult` | Winner announcement |
| `/play` | `PlayerJoin` | Enter PIN and name (pre-filled from email) |
| `/play/wait` | `PlayerWaiting` | Show team assignment, wait for host |
| `/play/game` | `PlayerGamepad` | 4-button answer controller (AnswerPad) |
| `/play/result` | `PlayerResult` | Win/lose screen on student device |

Pages use `useEffect` navigation guards to redirect if accessed without valid game state.

### Realtime Broadcast Events

| Event | Direction | Trigger |
|-------|-----------|---------|
| `player_join` | player → host | `joinRoom()` |
| `player_answer` | player → host | `submitAnswer()` |
| `game_start` | host → players | `startGame()` |
| `next_question` | host → players | `nextQuestion()` |
| `score_update` | host → players | `nextQuestion()` |
| `game_end` | host → players | `nextQuestion()` when out of questions |

### Key Components

- **`TimerBar`** (`src/components/host/`) — countdown timer (default 10s), calls `onExpire` when done; color shifts green → amber → red. Uses `onExpireRef` to avoid stale closure. Keyed to `currentQuestion.id` to reset on question change.
- **`KetupatAnim`** (`src/components/host/`) — horizontal tug-of-war; ketupat marker translates based on `scoreRendang - scoreOpor`, clamped to ±50.
- **`AnswerPad`** (`src/components/player/`) — 2×2 grid of giant buttons; shows correct/wrong color feedback after selection, then disables all buttons. Resets when `currentQuestion.id` changes.
- **`LoginForm`** (`src/components/shared/`) — magic link email form.
- **`ProtectedRoute`** (`src/components/shared/`) — renders children if authenticated, else renders `LoginForm`.

### Data

`src/data/mockSoal.json` — static question bank. Schema:
```json
{ "id", "tipe": "penjumlahan|pengurangan|perkalian", "difficulty": "mudah|sedang|sulit", "soal", "opsi": {"A","B","C","D"}, "jawaban_benar" }
```
27 questions — 3 per mode+difficulty combination (9 combos). Filtered by `gameConfig.mode` (→ `tipe`) and `gameConfig.difficulty`. Falls back to all questions if no match.
